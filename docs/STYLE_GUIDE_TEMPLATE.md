- Use active voice

- Maintain consistent tone and voice
- Apply consistent structure

- Keep information up-to-date
- Cite sources when appropriate
- Test examples and procedures

### Accessibility
- Use inclusive language
- Provide alternative text for images
- Use proper heading hierarchy
- Ensure sufficient color contrast

## Formatting Standards

### Headings
```markdown
# H1 - Main Title
## H2 - Section Title
### H3 - Subsection Title
#### H4 - Sub-subsection Title
```

### Lists
- Use bulleted lists for unordered items
- Use numbered lists for ordered steps
- Keep list items parallel in structure
- Use consistent punctuation in lists

### Code Blocks
```markdown
```language
# Example code with syntax highlighting
```
```

### Tables
- Use tables for structured data
- Keep tables simple and readable
- Align columns for readability
- Include table headers

### Links
- Use descriptive link text
- Link to relevant resources
- Avoid generic text like "click here"
- Check that links work

### Images
- Include alt text for all images
- Use appropriate image formats
- Optimize image file sizes
- Provide captions when needed

## Content Structure

### Document Organization
- Start with a clear purpose statement
- Use logical section organization
- Include a table of contents for long documents
- End with next steps or related resources

### Section Structure
- Begin sections with clear headings
- Use short paragraphs
- Include examples and illustrations
- End sections with key takeaways

### Examples and Tutorials
- Include clear learning objectives
- Provide complete, working examples
- Explain expected outcomes
- Include troubleshooting tips

## Visual Elements

### Diagrams and Charts
- Use clear, high-contrast visuals
- Include descriptive captions
- Provide alternative text descriptions
- Ensure accessibility compliance

### Screenshots
- Use current screenshots
- Highlight important elements
- Include descriptive captions
- Maintain consistent styling

### Icons and Symbols
- Use consistent iconography
- Provide text alternatives
- Follow accessibility guidelines
- Maintain visual hierarchy

## Technical Documentation

### API Documentation
- Include clear endpoint descriptions
- Document all parameters and responses
- Provide example requests and responses
- Include error handling information

### Code Examples
- Use real, working code examples
- Include necessary setup instructions
- Explain key concepts in examples
- Test all code examples

### Configuration Documentation
- Document all configuration options
- Include default values
- Explain the impact of changes
- Provide example configurations

## Accessibility Standards

### WCAG Compliance
- Follow WCAG 2.1 AA standards
- Ensure keyboard navigation
- Provide sufficient color contrast
- Include alternative text

### Screen Reader Compatibility
- Use proper heading hierarchy
- Provide link context
- Include form labels
- Ensure table accessibility

### Cognitive Accessibility
- Use clear, simple language
- Provide consistent navigation
- Minimize distractions
- Include help and support

## Localization Considerations

### Internationalization
- Use culturally neutral language
- Avoid idioms and colloquialisms
- Consider text expansion and contraction
- Support right-to-left languages

### Translation
- Write for translation
- Use consistent terminology
- Avoid complex sentence structures
- Include translation notes
